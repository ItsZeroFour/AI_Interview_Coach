# InterviewAI — AI-тренажёр собеседований

Платформа для подготовки к собеседованиям с адаптивным AI-агентом. Пользователь загружает резюме и описание вакансии — ИИ проводит персонализированное mock-интервью, оценивает ответы и даёт развёрнутую обратную связь.

## Project Brief

| Поле | Описание |
|------|----------|
| **Название проекта** | InterviewAI — AI-тренажёр собеседований |
| **Проблема** | Кандидаты не могут качественно подготовиться к собеседованиям: нет персонализированной обратной связи, вопросы из интернета не учитывают конкретную вакансию и опыт, а тренировка с живым человеком стоит дорого и не всегда доступна. |
| **Целевая аудитория** | Соискатели (junior–middle), готовящиеся к собеседованиям в IT и смежных областях. Сценарий: пользователь хочет потренироваться перед реальным собеседованием на конкретную позицию. |
| **Пользовательский сценарий** | 1) Загружает резюме (PDF/DOCX) и вставляет описание вакансии → 2) AI анализирует профиль и генерирует адаптивные вопросы → 3) Проходит mock-интервью в чате (5–8 вопросов) → 4) Получает детальный отчёт с оценкой по компетенциям и рекомендациями |
| **Какую задачу решает ИИ** | Генерация контекстных вопросов на основе резюме + вакансии, адаптация сложности по ходу интервью, оценка ответов с chain-of-thought рассуждением, генерация структурированной обратной связи |
| **Почему без ИИ решение хуже** | Без ИИ возможны только статичные списки вопросов без учёта конкретного профиля кандидата и вакансии. Нет адаптивности, нет персонализированной оценки, нет масштабируемости. |
| **Тип AI-адаптации** | AI-агент (agentic workflow) + продвинутый prompt engineering |
| **Baseline-решение** | Единичный вызов LLM с промптом «проведи собеседование» без контекста резюме/вакансии, без адаптации сложности, без структурированной оценки |
| **Целевые метрики** | Win-rate (improved vs baseline) ≥ 70%, Relevance score ≥ 4/5, Format compliance ≥ 90%, Latency < 5s на вопрос, Cost = $0 (free tier) |
| **MVP (must-have)** | Загрузка резюме, ввод вакансии, адаптивное интервью в чате (5–8 вопросов), оценка ответов, итоговый отчёт с аналитикой |
| **Риски** | Лимиты бесплатного API Gemini; качество парсинга резюме; галлюцинации модели при оценке; недостаток данных для eval-set |

## Стек технологий

- **Frontend:** React + TypeScript + SCSS, Vite

## Запуск проекта

### Требования

- Node.js 18+
- Python 3.11+
- Google Gemini API ключ (бесплатный, [получить здесь](https://aistudio.google.com/apikey))

### Backend

```bash
cd src/backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
# Вписать GEMINI_API_KEY в .env
uvicorn app.main:app --reload --port 8000
```

### Frontend

```bash
cd src/frontend
npm install
npm run dev
```

Приложение откроется на `http://localhost:5173`
