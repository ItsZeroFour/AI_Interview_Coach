# Чек-лист безопасности

| Категория | Что проверить | Статус | Комментарий |
|-----------|---------------|--------|-------------|
| **API-ключи** | Ключ Groq не попадает в git | ✅ | .env в .gitignore |
| | Ключ не отдаётся на фронтенд | ✅ | Все вызовы AI через backend |
| **Загрузка файлов** | Ограничен размер файла (макс 10 MB) | ✅ | MAX_FILE_SIZE_MB в .env |
| | Проверяется тип файла (только PDF/DOCX) | ✅ | ALLOWED_EXTENSIONS в upload.py |
| | Файлы не исполняются на сервере | ✅ | Только парсинг текста |
| | Файлы удаляются после парсинга | ✅ | finally: filepath.unlink() |
| **Prompt injection** | Системный промпт не раскрывается пользователю | ✅ | Промпт в серверном коде |
| | Пользовательский ввод ограничен по длине | ✅ | max_length=5000 в AnswerRequest |
| | Есть фильтрация опасного ввода | ⚠️ | Частично - через Pydantic валидацию |
| **Rate limiting** | Ограничено кол-во интервью с одного IP | ❌ | Не реализовано в MVP |
| | Ограничена длина ответа пользователя | ✅ | max_length=5000 |
| **Данные** | Резюме не хранятся дольше сессии | ✅ | In-memory, удаляются при перезапуске |
| | Пользователь предупреждён об использовании AI | ⚠️ | Указано на лендинге, нет явного consent |
| **Фильтрация вывода** | AI не генерирует оскорбительный контент | ✅ | Системный промпт задаёт рамки |
| | Fallback при ошибке AI | ✅ | AnswerEvaluation с дефолтными значениями |
| **CORS** | Настроен белый список origins | ✅ | ALLOWED_ORIGINS_RAW в .env |
| **Инференс** | Обработка ошибок API (timeout, 429) | ✅ | try/except + fallback модель |
| | Логирование ошибок | ✅ | logging в agent.py и interview.py |
